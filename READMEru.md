## Что такое *SSE*?
***SSE (Server Send Event)*** - это способ передачи сообщений от сервера к клиенту. По умолчанию сервер не может отправлять сообщения клиенту, поскольку не распознает его. Однако клиент может легко получить доступ к серверу, т.к. у сервера есть публичный IP-адрес.

## Зачем использовать *SSE*?
Если мы хотим, чтобы сервер имел возможность отправлять сообщения клиенту.

## Когда следует использовать *SSE*?
Использовать *SSE* удобно, когда нам требуется простая односторонняя связь только между сервером и клиентом. Если же необходима двусторонняя связь между сервером и клиентом, то лучше использовать *WebSocket*.

## Как работает система *SSE*?
Как говорилось, сервер не распознает клиента, первое рукопожатие всегда должно начинаться клиентом (**1**). После получения запроса на рукопожатие от клиента (*TLS handshake*)  сервер не отвечает сразу, а удерживает ответ.

## Что это значит?
Непосредственно перед тем, как сервер должен вернуть ответ, он "перехватывает" этот запрос в бесконечном цикле. В условиях такого "вечного" бесшовного соединения сервер может использовать преимущество отправки клиенту любого желаемого количества сообщений (**2, 3, 4**). Фактически, в этом случае клиент бесконечно ждет завершения ответа сервером, но этого никогда не происходит, пока сервер не решит завершить его (**5**).

## Как разорвать соединение?
Сервер может закрыть соединение с клиентом, просто выйдя из цикла. С другой стороны, клиент также может легко закрыть соединение с сервером, отменив запрос на установление соединения, который не был завершен (из-за цикла). Это также заставит сервер закрыть соединение с клиентом.

## Почему просто не использовать *WebSocket*?
***WebSocket*** - двухсторонняя связь между сервером и клиентом
***SSE*** - односторонняя связь между сервером и клиентом
Можно также использовать *WebSocket*, но это будет чуть медленней по *RPS*, и сложнее реализовать, чем *SSE*.

## Как реализовать *SSE*-сервер на Go?
Мы будем использовать встроенную библиотеку ***net/http*** для работы с *HTTP*. Можно использовать другие библиотеки, такие как *gin* или *echo*.
```go
func (w http.ResponseWriter, r *http.Request)
```
Создайте канал с именем *messageChan*. Сообщение клиенту будет доставлено через него. Переменная канала должна быть определена где-нибудь, где мы сможем к ней получить доступ. Допустим, сейчас мы просто отправляем строковое сообщение.
```go
messageChan = make(chan string)
```
Для части кода, отвечающей за "trap-request", мы будем использовать бесконечный цикл. Внутри этого цикла процесс будет заблокирован каналом messageChan. Канал messageChan продолжает ждать и прослушивать готовность сообщения. Если сообщение доступно, оно будет отправлено в HTTP-записывающее устройство. Нам также необходимо отправить сообщение в кэш, чтобы клиент мог его увидеть. В случае получения сигнала о закрытии соединения от клиента, функция `Context().Done()` выдаст сигнал "close", и мы сможем выйти из цикла (фактически, мы выходим их функции, а не просто из цикла). Здесь мы поддерживаем соединение.
```go
rc := http.NewResponseController(w)

for {
	select {
		case message := <- messageChan:
			fmt.Fprintf(w, "data: %s\n\n", message)
			rc.Flush()
		case <-r.Context.Done():
			return
	}
}
```
Непосредственно перед выходом их функции необходимо закрыть канал, чтобы избежать утечки памяти. Это можно сделать через `defer`.
```go
defer func() {
	close(messageChan)
	messageChan = nil
}()
```
## Как начать отправку сообщения клиенту?
Мы можем просто поместить сообщение в канал. Убедитесь, что канал `messageChan` был создан до этого.
```go
messageChan <- "Привет"
```

## Как реализовать SSE-клиент на Go?
Сначала мы инициализируем HTTP-клиент и подготавливаем API для установления соединения с сервером.
```go
client := new(http.Client)
req, _ := http.NewRequest("GET", "http://localhost:3000/handshake", nil)
```
Начинаем с вызова сервера. Если ответа от сервера нет, продолжаем попытки. Здесь мы используем бесконечный цикл. Мы называем это состояние `TRY_OPEN_CONNECTION`. Если ошибок не возникает, это означает, что соединение успешно установлено.
```go
for {
	// TRY_OPEN_CONNECTION состояние
	res, err := client.Do(req)
	if err != nil {
		time.Sleep(time.Second)
		continue
	}
	log.Println("соединение установлено..")
	
	...
}
```
Продолжая тот же цикл, нам нужно обработать следущее состояние `WAITING_MESSAGE`. Нам нужен ещё один цикл для поддержания процесса получения сообщений от сервера.
```go
for {
	// TRY_OPEN_CONNECTION состояние
	
	// WAITING_MESSAGE состояние
	stillListening := true
	for stillListening {
		...
	}
}
```
Мы объявляем два канала: 
```go
messageChan := make(chan string) // для сообщений
stopChan := make(chan int) // для остановки цикла
```
Мы определяем горутину и немедленно запускаем её, чтобы прослушать сообщение. Любое сообщение будет отправлено в `messageChan`, а любая ошибка (`connection closed`) запустит канал `stopChan`.
```go
go func() {
	scanner := bufil.NewScanner(res.Body)
	
	if scanner.Scan() {
		messageChan <- scanner.Text()
	}
	
	if err := scanner.Err(); err != nil {
		stopChan <- 1
	}
}()
```
И, наконец, последний шаг - выбор канала, который будет активирован. Функция stopChan изменит состояние `stillListening` (что прервет цикл ожидания состояния `WAITING_MESSAGE`). С другой стороны, `messageChan` просто выведет сообщение в консоль.
```go
select {

	case <- stopChan:
		stillListening = false
		
	case message := <- messageChan:
		log.Println(message)
}
```
